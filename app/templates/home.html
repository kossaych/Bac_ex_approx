During the BAC algorithms exam, I encountered a grading issue in exercise 4 (approximation problem), which specifically involved estimating the nth root. This task required developing a method to estimate the nth root of a number, involving complex calculations (over 1000 steps), making it impossible for the grader to verify the method's validity if the proposed solution differed from the official one.

I proposed a different solution than the one in the official correction. My solution worked well when I tested it using Python, and its effectiveness was confirmed by my high school teacher. It succeeded in estimating the nth root.

However, during the grading process, there was a misunderstanding. The exercise was graded manually, which is unsuitable for such complex tasks (over 1000 steps). My teacher explained that, in BAC exams, for approximation exercises like this, graders do not strictly check each step or the final results because they cannot compute them manually; they need a computer. Instead, the evaluation focuses on the approach and logic due to the complexity involved. This approach can be unfair to creative and unconventional students like me because my proposed solution was different.

    def estpremier(x) :
        i = 2
        test = True
        while(test == True and i<= x//2) :
            if x%2 == 0 :
                test = False
            i = i + 1
        return test
    def premier_suivant(n): 
        n += 1
        while not estpremier(n):
            n += 1
        return n
    def Facteur(M) : 
        ch = ""
        while M > 1:
            i = 2
            while M % i != 0:
                i = premier_suivant(i)
            ch = ch  + str(i)
            M = M // i
        max = int(ch[0])
        for i in range (0,len(ch)) :
            if int(ch[i]) > max :
                max = int(ch[i])
        return max
    def suitex(M,n,k) :
        if k == 0 :
            return M/2
        else :
            Tp = suitex(M,n,k-1)
            return (1 / n) * ((n - 1) * Tp + (M / (Tp ** (n - 1)))) 
      
    def RacineN(M,n) :
        Tp = suitex(M,n,1)
        T = suitex(M,n,2)
        k = 2
        while(abs(Tp - T) > 0.0000000000000000000000000001) :
            Tp = T
            k = k + 1
            T = suitex(M,n,k)  
        return T
    def NRound(M) :
        n = 2
        p = Facteur(M)
        while(p < RacineN(M,n) ) : 
            n = n+1
        return n - 1
